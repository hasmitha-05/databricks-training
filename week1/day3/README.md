# Employee SQL Queries

This repository contains SQL queries performed on the `Employees` table using different SQL clauses and operators. These queries help in understanding how to retrieve, filter, sort, and analyze employee data in a database.

---

# Table Used

The Employees table contains the following columns:

- emp_id
- emp_name
- department
- salary
- city
- experience

---

# TOP / LIMIT

The TOP or LIMIT clause is used to display a limited number of records from a table. It helps in retrieving only the required rows such as highest salaries or most experienced employees.

# DISTINCT

The DISTINCT keyword is used to display unique values from a column by removing duplicate records.

# Comparison Operators

Comparison operators are used to compare values in SQL queries.

Operators:
- >
- <
- >=
- <=
- <>

These operators are mainly used in the WHERE clause to filter records based on conditions.


# Logical Operators

Logical operators are used to combine multiple conditions in a query.

Operators:
- AND
- OR
- NOT

These operators help in applying multiple conditions together.

# IN and NOT IN

The IN operator is used to match multiple values in a condition.  
The NOT IN operator is used to exclude specific values from the result.


# BETWEEN Operator

The BETWEEN operator is used to select values within a specified range. It is commonly used with salary, experience, and numeric values.


# LIKE Operator

The LIKE operator is used for pattern matching in SQL. It helps in searching records based on specific text patterns.

Wildcard:
- `%` → Represents multiple characters


# Conclusion

These SQL queries cover important concepts such as filtering records, sorting data, finding unique values, pattern matching, and applying conditions using operators. Practicing these queries improves SQL skills and helps in understanding real-time database operations effectively.
